# 从多跳检索到潜在推理：构建“思考后再检索”的 RAG 系统

## 1. 问题背景：为什么需要多步检索？

传统的 RAG 系统采用单步检索：给出查询，从知识库中检索 top‑k 片段，拼接后交给 LLM 生成答案。这一模式在简单问题上效果良好，但在以下场景中会失效：

- **多跳推理**：答案需要串联多个文档中的证据才能得出（如“作者 A 的导师 B 是哪一年获得诺贝尔奖的？”）。
- **时间顺序推理**：需要按照事件发生的先后顺序梳理线索。
- **长上下文证据链搜索**：关键证据散落在超长文档的不同位置，需要逐步定位。

这些任务的本质是**序列决策问题**——第一步找到的内容会影响第二步应该找什么，第二步又会影响第三步。单次相似度排序无法解决这种“铺路”需求。

## 2. 现有方案对比

### 2.1 训练 LLM 学会搜索（Agentic RAG）

让 LLM 显式地生成子查询、调用检索工具、观察结果并决定下一步。代表方法包括 Self‑RAG、ReAct、AutoGPT 等。

- **优点**：灵活，可解释性强。
- **缺点**：每一步都要生成 token、调用外部工具，延迟高；且需要 LLM 具备强大的遵循指令和推理能力，通常要用 RL 微调 LLM，成本极高。

### 2.2 Q-RAG：训练检索器学会多步搜索

Q-RAG（ICLR 2026 Oral）的思路与主流相反：**冻结 LLM，训练嵌入器（retriever）学会多步决策**。它将多步检索建模为 MDP，用 soft Q‑learning 训练两个嵌入器分别编码“状态”（已有证据）和“动作”（候选片段），通过内积近似 Q 函数，选出每一步价值最高的片段。

- **优点**：检索器轻量，训练成本低（单张 A100），可与任意 LLM 配合，推理快，在多跳和长上下文任务中表现优异。
- **局限**：依赖支持事实作为奖励监督，实际应用中可能缺少如此干净的标注；仍然需要多次显式检索。

## 3. 我们的初步构建：rerank + highlight 驱动的多步检索

在阅读 Q-RAG 之前，我们已经尝试了一种朴素的迭代检索方案：

1. 用 LLM 的隐藏状态作为查询向量进行检索；
2. 检索到的片段经 **reranker** 精排、**highlight** 分析（评估与已有证据的衔接性、信息增益等）；
3. 将最佳片段拼接到查询文本中，形成新的查询；
4. 重复上述过程，直到收集到足够的证据链。

这种方法本质上是用手工特征（rerank 分数 + highlight 指标）近似 Q-RAG 中的 `Q(state, action)`，相当于一个**无训练的多步检索价值评估系统**。在实践中它能显著改善多跳推理的质量，但也面临计算开销大、误差累积等问题。

## 4. 深层洞察：能否把“多步决策”内化到 LLM 的潜在空间？

上述流程无论怎样优化，仍需在 token 空间显式地生成查询、检索、拼接，占用大量上下文窗口且速度受限。一个自然的念头是：

**让模型不在表层生成查询文本，而是在隐藏状态空间里“安静地思考多步”，将推理过程压缩成一个向量，再直接用这个向量进行检索。**

这相当于把 agent 的多跳检索动作，蒸馏到 LLM 的前向传播中。

### 4.1 关键技术：自回归潜思维生成（LatentMAS）

LatentMAS（以及更早的 Coconut）证明了一件事：

- 传统推理：模型生成 token，再送入输入层。
- 潜在推理：模型取最后一层的隐藏状态，通过一个可训练的投影矩阵，直接作为下一轮的输入嵌入，**不生成任何 token**。
- 经过若干步这样的隐式循环，模型能完成多步推理任务，且比显式思维链（CoT）效率更高、泛化能力更强。

这为我们的设想提供了技术基础：**可以进行 50 步甚至更多的潜在推理，得到富含推理历史的深层表示。**

## 5. 完整构想：潜在推理检索系统

### 5.1 系统流程

1. **输入问题**，嵌入为初始隐藏状态。
2. **在潜在空间循环思考**：自回归地进行 *T* 步隐藏状态更新（例如 50 步），每一步的隐藏状态都作为下一步的输入嵌入。
3. **输出最终隐藏状态**，这个向量被认为融合了解决该问题所需的多步推理链路。
4. **将最终隐藏状态映射到检索空间**：通过一个训练好的投影层，将其转换为与文档向量处于同一度量空间的查询向量。
5. **单次检索**：使用该查询向量在文档向量库中做最近邻搜索，直接得到最终所需的证据片段。
6. **拼接证据，送入 LLM 生成答案**（或直接由潜在状态解码出答案）。

### 5.2 为什么这个想法是突破性的？

| 维度 | 当前多跳检索 | 我们的潜在推理检索 |
|------|-------------|-------------------|
| 生成 token | 每一步都要生成新查询/关键词 | 零 token 生成 |
| 检索次数 | 多次（等于推理步数） | **一次** |
| 上下文窗口 | 证据累积会撑爆窗口 | 仅最终检索片段进入上下文 |
| 推理与检索的计算 | 交替进行，串行阻塞 | 潜在推理一步到位，检索并行友好 |
| 学习信号 | 需要人工模板或复杂 RL | 可利用 rerank+highlight 作为监督 |

## 6. 核心挑战与解决路径

潜在推理得到的隐藏状态**不会天然适合检索**，因为 LLM 的隐藏空间是为预测下一个 token 优化的，缺乏对比结构（相似查询的向量未必靠近）。因此，我们需要额外的训练来对齐两个空间。

### 6.1 添加检索投影层（Retrieval Projection）

在最终隐藏状态之上，增加一个可训练的投影矩阵 *W*，将其映射到与文档嵌入相同的空间。该文档嵌入空间由同一个 LLM（或专用嵌入模型如 E5‑Mistral）对文档进行编码得到。

### 6.2 使用对比学习训练投影层

正例：问题相关的文档。  
负例：不相关的文档或随机采样文档。  
损失函数：使查询向量与正例文档向量的余弦相似度尽可能高，与负例尽可能低。

这能强迫模型学会：**通过内部潜在推理，将自己的隐藏状态调整到一个“能检索到正确证据”的方向上。**

### 6.3 利用现有 rerank + highlight 流程提供监督

我们的 rerank + highlight 流程天生适合作为这个训练的信号源：

- 对于一批训练问题，用潜在推理后的向量检索候选片段；
- 用 reranker 和 highlight 模块评估这些片段的质量（是否真正有助于回答问题）；
- 将 rerank 分数作为软标签或奖励，更新投影层（甚至微调潜在推理部分的参数），使模型的“思考结果”更倾向于检索到高价值片段。

这相当于用我们自己系统的“判断力”来蒸馏出一个高效的潜在检索模型，实现 **RL‑free 的多跳检索内化**。

## 7. 与 Q-RAG 的关系：更进一步的范式

- Q-RAG 仍然需要**显式的多步检索动作**（在文档集合中选择 chunk），只是决策网络被训练得很好。
- 我们的方案是将这个**显式决策过程完全折叠进 LLM 的潜在空间**，最后只需一次检索。如果说 Q-RAG 是“训练检索器像 agent 一样搜索”，我们则是“训练 LLM 在思考中直接抵达答案的文档邻域”。

两者的共同点在于都把多跳推理的优化对象从 LLM 生成转移到了检索/嵌入层，但我们更进一步压缩了搜索的时空开销。

## 8. 总结与展望

本文记录了一条从“朴素迭代检索”到“潜在推理检索”的完整思路演进：

1. 发现单步 RAG 的局限，尝试迭代检索；
2. 引入 rerank + highlight 模拟 Q‑value 选择；
3. 受 Q-RAG 和 LatentMAS 启发，提出将多跳决策内化到 LLM 的潜在空间；
4. 设计出“潜在循环思考 → 单次检索”的高效框架，可利用已有 rerank 流程作为学习信号。

这一方案如果实现，将大幅提升多跳 RAG 的效率与集成度，同时保持极低的推理成本。我们相信，**让模型“想清楚再搜”，而不必“边想边搜”，是下一代检索增强生成的重要方向。**
